# WorkLedger-for-Staffing
**人材派遣管理システム（業務・法令・利益を一元管理するインテリジェンス・ボード）**

WorkLedger は、人材派遣業における  
**労働（Work）とお金（Ledger）を一元管理し、意思決定を加速させる業務管理システム**です。

契約・勤怠・36協定・利益分析といった  
**分断されがちな管理情報を、1つのダッシュボードに統合**することを目的に個人開発しています。

---

## 🔍 背景・課題

人材派遣業では、以下のような情報を同時に管理する必要があります。

- スタッフ・案件・アサイン状況  
- 契約更新期限  
- 勤怠・残業時間・36協定  
- 請求・給与・粗利  
- 月次・年次の利益分析  

多くの現場では、  
Excel や複数の管理ツールに情報が分散し、

- 利益は出ているが、どこにリスクがあるか分からない  
- 契約更新や36協定の管理が後手に回る  
- 判断のために毎回集計が必要  

といった課題が発生します。

**WorkLedger は「記録するためのシステム」ではなく、  
「判断するためのシステム」**として設計しています。

---

## ✨ 主な機能・設計思想

### 📊 意思決定を加速するダッシュボード
- 売上・粗利・マージン率の可視化  
- 労働時間と利益の相関分析  
- 一人当たり生産性指標  

### ⏰ 契約更新・法令リスクのアラート管理
- 契約更新期限の可視化・警告  
- 残業時間・36協定リスクの早期検知  

### 👥 人材派遣業務の一元管理
- スタッフ・案件・アサイン管理  
- 勤怠・給与・請求データの集約  

> ※ 画面・機能はすべて **業務モデルの例示（サンプル）**です。  
> 実在の企業・契約・個人情報とは一切関係ありません。

## 🧩 技術構成（Tech Stack）

- **Frontend** : Next.js / Tailwind CSS  
- **Backend**  : NestJS  
- **Database** : MySQL  
- **Runtime**  : Node.js（TypeScript）

業務ロジックと UI を明確に分離し、  
月次集計・法定帳票・将来的な拡張を前提とした構成です。
